使用 Django-Celery + Redis 实现定时任务完整指南

背景需求

在调用（DeepSeek）API的服务时，我们为每个用户设置了每日50次的API调用限额。为了实现每日零点的自动重置功能，需要构建可靠的定时任务系统。

技术选型对比

方案 优势 劣势 适用场景
Linux Crontab 系统级支持，配置简单 与Django环境隔离，日志管理不便 简单脚本任务
APScheduler 纯Python实现，轻量级 缺乏分布式支持，可靠性较低 单机简单定时任务
Celery + Redis 分布式支持，任务可追溯，集成Django 需要额外中间件，配置较复杂 生产环境可靠定时任务（本文选择）

最终选择组合：

  • **Celery 5.3+**：分布式任务队列
  • **Redis 6+**：消息代理 & 结果存储
  • **django-celery-beat 2.5+**：基于数据库的调度器
  • **Supervisor 4.2+**：进程守护
