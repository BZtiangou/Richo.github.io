Read only memory

只读存储器（Read Only Memory，ROM）是一种存储器类型，用于存储数据和指令，其中存储的内容在制造过程中被预设并无法被修改。ROM中的数据只能被读取，而不能被写入或擦除。

点标记法（dot notation）:

以下是使用点标记法（dot notation）来表示只读存储器（ROM）的内容，并说明ROM中电流的流动方向:

  1. “The contents of a ROM can be indicated using dot notation.”（可以使用点标记法来表示ROM的内容。）： 这句话意味着可以使用点标记法来表示ROM中存储的数据内容。点标记法是一种表示二进制数据的方法，其中用点（.）表示二进制位的值。例如，一个4位的ROM中，”1101”可以表示为”.1..”, 其中每个点对应一个二进制位的值。
  2. “Here, the dots indicate the data. It is not means the lines are simply connected.”（这里，点表示数据，它并不意味着线路仅仅是连接的。）： 这句话强调了点标记法中的点仅表示数据的值，而不是简单地表示线路的连接。换句话说，点标记法中的点表示ROM中存储的数据位的值，而不是仅仅表示线路是否连接。这是因为ROM中的数据是通过存储元件（如触发器或闪存电容）实现的，数据存储的方式不仅涉及线路连接，还涉及物理元件的状态。
  3. “The electric current can only flow from the wordlines to bitlines.”（电流只能从字线流向位线。）： 这句话指的是在ROM的操作中，电流只能从字线（wordline）流向位线（bitline）。字线是用于选择存储阵列中的特定行的线路，而位线是用于读取和写入特定位单元中的数据的线路。在ROM的读取操作中，电流从被选中的字线流向位线，以读取相应位单元中存储的数据。写入操作时，字线和位线之间的电流用于将数据写入位单元。

单处理器

在单个周期(在一个CLK周期内)执行指令。对于最慢的指令，CLK周期必须足够长。

Multicycle Processor

The multicycle processor breaks an instruction into multiple shorter stepsby adding a group of registers.Thus, some instructions require 5 cycles, but other only require 4 cycles.

多周期的处理器
多周期处理器通过添加一组寄存器将一条指令分解成多个较短的步骤因此，有些指令需要5个周期，而其他指令只需要4个周期。

管线式处理器

管线式处理器五个指令可以同时执行，每个阶段一个。流水线使用寄存器将单周期数据路径切割为5段。因此，这些部分可以独立于其他部分工作。所以每个部分后面都需要加上一个寄存器文件储存输入。

DMA（直接内存访问）

DMA（直接内存访问）是计算机体系结构中的一种技术，它允许外部设备直接与计算机的内存进行数据传输，而无需CPU的干预。通过使用DMA，外设可以高速地传输大量数据，而无需将数据逐个字节地通过CPU进行处理和复制，从而提高了系统性能和效率。

在传统的计算机架构中，当外部设备（如硬盘驱动器、网络适配器、显卡等）需要与内存进行数据交换时，必须通过CPU来完成。这种方式称为程序控制I/O（Programmed I/O）。在程序控制I/O中，CPU将从外设读取或写入数据的任务完全交给自己来处理，这意味着CPU需要逐个字节地传送数据，从而占用了大量的CPU时间和计算资源。

而DMA技术的引入则改变了这种情况。DMA控制器是一种专门的硬件设备，它具有自己的地址总线、数据总线和控制总线。DMA控制器可以直接访问系统内存，并与外部设备进行高速数据传输，而无需CPU的干预。

DMA的工作过程如下：

  1. 外设向DMA控制器发送数据传输请求。

  2. DMA控制器与CPU进行协调，请求获得总线的控制权。

  3. CPU授权DMA控制器访问系统总线。

  4. DMA控制器通过总线直接访问内存，并与外设进行数据传输。

  5. 数据传输完成后，DMA控制器释放总线的控制权，通知CPU传输结束。

通过使用DMA，数据可以在外设和内存之间直接传输，而无需CPU的频繁干预。这可以大大提高数据传输速度和系统效率，同时减轻CPU的负担，使CPU能够更多地专注于处理其他重要的计算任务。

需要注意的是，DMA仍然需要CPU的参与来设置和管理数据传输的参数，例如传输的起始地址、传输长度等。此外，为了确保数据的一致性和正确性，DMA控制器和CPU之间需要进行适当的同步和通信。

总而言之，DMA是计算机中的一种技术，它允许外设直接与内存进行数据传输，提高了数据传输速度和系统效率，减轻了CPU的负担。

Port-mapped I/O（端口映射I/O）

Port-mapped I/O（端口映射I/O）是一种用于访问设备的技术，它使用额外的地址空间（即端口）来与设备进行通信。

在计算机中，内存和I/O设备被分配了不同的地址空间。内存通常使用内存地址进行寻址和访问，而I/O设备则使用不同的I/O地址进行寻址和访问。在端口映射I/O中，为了访问I/O设备，计算机体系结构会提供一组专门的I/O端口，这些端口是专门为与设备进行通信而预留的。

每个I/O设备都会被分配一个或多个独立的I/O端口。通过读取或写入这些特定的I/O端口，可以与相应的设备进行通信。与内存地址不同，I/O端口通常是一个较小的地址范围，通常以端口号来标识。

使用端口映射I/O时，CPU通过指定端口号来读取或写入设备的寄存器或数据。这些设备可能是输入设备（如键盘、鼠标）或输出设备（如显示器、打印机）。通过读写这些设备的寄存器，可以控制设备的操作和传输数据。

总结起来，端口映射I/O是一种用于访问设备的技术，它使用额外的地址空间（即端口）来与设备进行通信。通过读写特定的I/O端口，CPU可以控制设备的操作和传输数据。

Memory-mapped I/O（内存映射I/O）

Memory-mapped I/O（内存映射I/O）是一种用于访问设备的技术，它将I/O设备映射到与主存储器相同的地址空间中。处理器可以通过读写指定的地址来访问设备进行数据的读取和写入。

在计算机中，主存储器（RAM）用于存储程序和数据，而I/O设备用于输入和输出数据。在内存映射I/O中，为了访问I/O设备，计算机体系结构将I/O设备的寄存器或缓冲区与主存储器的地址空间中的特定地址相关联。

使用内存映射I/O时，设备的寄存器或缓冲区被映射到主存储器的一部分地址范围内。处理器可以像访问主存储器中的数据一样，通过读写相应的地址来访问设备的寄存器或缓冲区。

例如，假设一个设备的寄存器被映射到主存储器的地址范围0x1000到0x100F。处理器可以使用读操作从地址0x1000读取设备的状态信息，或者使用写操作将数据写入地址0x1004以控制设备的操作。在处理器的视角中，这些地址看起来像是主存储器中的普通内存地址，但实际上它们对应于设备的寄存器或缓冲区。

通过使用内存映射I/O，设备可以像访问内存一样进行读写操作，使得设备的访问更加统一和简化。处理器无需使用专门的I/O指令来访问设备，而是直接通过内存地址来进行操作。

总结起来，内存映射I/O是一种将I/O设备映射到与主存储器相同的地址空间中的技术。处理器可以通过读写指定的地址来访问设备进行数据的读取和写入。通过使用内存映射I/O，设备的访问方式更加统一和简化，处理器可以直接通过内存地址来操作设备。

FP,GP,RA寄存器：

以下是对GP（General Purpose，通用寄存器）、FP（Frame Pointer，帧指针）和RA（Return Address，返回地址）寄存器的详细介绍：

  1. GP寄存器（General Purpose，通用寄存器）：

    • 通用寄存器是一组用于存储通用数据和中间结果的寄存器。它们被设计为具有多种用途，可根据需要用于不同的计算和操作。
    • 通常，通用寄存器以编号或名称表示，例如R0、R1、R2等。它们在程序执行期间可以用于存储临时变量、函数参数、返回值和其他临时数据。
    • 通用寄存器的数量和位宽度取决于计算机架构和处理器的设计。常见的架构中，通常有一组固定数量的通用寄存器可供程序员使用。

  2. FP寄存器（Frame Pointer，帧指针）：

    • 帧指针是一个特殊的寄存器，用于在函数调用期间管理函数的活动记录（activation record）或称为帧（frame）。
    • 在函数调用时，帧指针用于指示当前函数的堆栈帧的起始位置。堆栈帧是一个包含函数局部变量、参数和其他与函数执行相关的信息的区域。
    • 帧指针的值在函数调用期间保持不变，使得函数可以通过固定偏移量来访问局部变量和参数。它还可以用于恢复函数返回时的堆栈位置。

  3. RA寄存器（Return Address，返回地址）：

    • 返回地址寄存器用于存储函数调用之后要返回的地址。当一个函数调用另一个函数时，当前函数的返回地址会被保存在返回地址寄存器中。
    • 返回地址是指调用函数的指令的下一条指令的地址，即将控制权返回到调用函数后继续执行的位置。
    • 当被调用的函数执行完毕时，返回地址寄存器中的值被用来跳转回调用函数的下一条指令，从而实现函数调用的返回。

    SP寄存器

    SP寄存器指示了栈的当前顶部地址，也就是下一个将被弹出的数据的位置。当新数据被推入栈时，SP寄存器的值会减小，指向栈中的更高地址。当数据被弹出栈时，SP寄存器的值会增加，指向栈中的较低地址。因此，SP寄存器通常用于维护和跟踪栈的顶部位置，将栈顶的地址保存在SP寄存器中。所以，陈述是正确的。

可编程逻辑阵列（Programmable Logic Arrays，PLA）：

可编程逻辑阵列（Programmable Logic Arrays，PLA）是一种可编程逻辑器件，用于实现数字电路的逻辑功能。它是一种具有可编程输入和可编程输出的组合逻辑电路。

PLA由两个主要部分组成：

  1. AND阵列：它由多个可编程的与门组成。每个与门都有多个输入引脚，这些输入引脚可以通过编程进行连接或屏蔽。通过编程，可以确定哪些输入信号在与门中参与运算。
  2. OR阵列：它由多个可编程的或门组成。每个或门都有多个输入引脚，这些输入引脚与AND阵列的输出相连。通过编程，可以确定哪些AND门的输出将作为输入连接到每个或门。

PLA的编程是通过设置或烧录内部的开关或存储器单元来完成的。这些开关或存储器单元的状态决定了逻辑门之间的连接关系，从而实现了特定的逻辑功能。通过重新编程这些开关或存储器单元，可以改变PLA的逻辑功能。

PLA的主要优点是具有灵活性和可重构性。通过重新编程，可以改变PLA的逻辑功能，适应不同的应用需求，而无需更换硬件。这使得PLA成为实现逻辑功能的可编程解决方案之一。

PLA在数字系统设计中得到广泛应用。它提供了一种灵活和可定制的方式来实现复杂的逻辑功能，同时具有较低的成本和较短的开发周期。
