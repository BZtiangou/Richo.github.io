微信小程序开发

基本组构：

1.Pages文件夹用来存放所有的页面

2.app
3..json：用来存放配置信息，其”pages”:[xx]用来新增页面的存放路径，从根目录开始算起。

项目首页是pages中的第一个，同时它也是按前后顺序进行渲染。

3..js文件是逻辑交互，特别的，APP（）函数是小程序的启动函数

4.wxss是类似于CSS的一种样式表

5.json是一种数据格式，用来存写配置信息

6.sitemap.json 是用来微信索引

7.wxml 和 HTML的区别，标签不同，属性不同，提供了Vue中的模板语法

API

微信小程序的宿主环境是手机微信！

官方把API分成三类：

1.事件监听类：

事件监听 API
特点: 以 on 开头，用来监听某些事件的触发
举例: wx.onwindowResize(function callback) 监听窗口尺寸变化的事件同步

2.同步API：
特点1:以 Sync 结尾的 API 都是同步 API
特点2: 同步 API的执行结果，可以通过函数返值直接获取，如果执行出错会抛出异常举例: wx.setStorageSync(key’，value)向本地存储中写入内容

3 异步 API
特点: 类似于jQuery 中的 $.ajax(options)函数，需要通过 success、fail、complete 接收调用的结果举例: wx.request()发起网络数据请求，通过 success 回调函数接收数据

协同工作和发布

管理员是大boss：

小程序成员管理体现在管理员对小程序项目成员及体验成员的管理。

项目成员:
表示参与小程序开发、运营的成员
可登录小程序管理后台
管理员可以添加、删除项目成员，并设置项目成员的角色

体验成员:
表示参与小程序内测体验的成员

可使用体验版小程序，但不属于项目成员

管理员及项目成员均可添加、删除体验成员

我们大多是作为开发者

开发者权限:可使用小程序开发者工具及对小程序的功能进行代码开发
体验者权限:可使用体验版小程序
登录权限:可登录小程序管理后台，无需管理员确认
开发设置:设置小程序服务器域名、消息推送及扫描普通链接二维码打开小程序
腾讯云管理:云开发相关设置

数据绑定

在data中定义，在wxml中使用。使用的时候要使用Mustach(双大括号)语法

例如：Page({data:{info:”hello world”}})

事件：

渲染层到逻辑层的通信方式。

常用事件：tap,input,change

当事件回调发生时，收到一个事件对象event。

target是触发事件的组件的一些属性值集合

遇到js和ts文件怎么办？

  • ts是js的超集，即你可以在ts中使用原生js语法。

  • ts需要静态编译，它提供了强类型与更多面向对象的内容。

  • ts最终仍要编译为弱类型的js文件，基于对象的原生的js，再运行。故ts相较java/C#这样天生面向对象语言是有区别和局限的

  • ts是由微软牵头主导的，主要来自C# TypeScript和JavaScript的对比：

  • TypeScript是一个应用程序级的JavaScript开发语言。（这也表示TypeScript比较牛逼，可以开发大型应用，或者说更适合开发大型应用）

    所以ts直接当js用就行了，类似于C++和C

事件传参：

不能在绑定事件的同时为事件处理函数传递参数 VUE是可以的
