捕获异常篇：

异常处理是一种编程概念，用于处理程序执行过程中可能发生的错误或异常情况。当发生异常情况时，会引发一个异常，表示出现了意外或错误的情况。异常处理提供了一种结构化的方式来捕获和处理这些异常。

在大多数编程语言中，包括Python、Java和C++，使用try-catch块来进行异常处理。下面是它的一般工作流程：

可能引发异常的代码放置在try块中。

如果在try块中发生异常，就会抛出或引发该异常。

抛出的异常被一个或多个catch块捕获。

catch块通过执行特定的代码来处理异常情况。

在执行完catch块后，程序会从catch块后面的代码继续运行。

在异常处理中，try 块可以与其他关键字搭配使用，以实现更丰富的异常处理逻辑。以下是一些常见的与 try 搭配使用的关键字：

except: except 关键字用于捕获指定类型的异常，并定义相应的异常处理代码块。可以在 try 块后面跟随一个或多个 except 块，每个 except 块捕获特定类型的异常并处理。可以根据需要指定多个不同类型的异常。

else: else 关键字用于定义在 try 块中没有引发异常时要执行的代码块。如果在 try 块中没有发生任何异常，那么将执行 else 块中的代码。else 块是可选的。

finally: finally 关键字用于定义一个无论是否发生异常都会执行的代码块。无论异常是否被捕获和处理，finally 块中的代码都会执行。它主要用于执行一些必要的清理操作，例如关闭文件或释放资源。

文件操作：

主要想讲一下:

在 Python 的文件操作中，有多种打开文件的模式。下面列出了常见的文件打开模式及其用途：

  1. “r” (Read Mode):
    • 用途：以只读模式打开文件。
    • 文件指针位置：文件开头。
    • 如果文件不存在，会引发 FileNotFoundError 异常。
  2. “w” (Write Mode):
    • 用途：以写入模式打开文件，如果文件存在则清空文件内容，如果文件不存在则创建新文件。
    • 文件指针位置：文件开头。
    • 如果文件无法打开，会引发 PermissionError 异常。
  3. “a” (Append Mode):
    • 用途：以追加模式打开文件，如果文件存在则将内容添加到文件末尾，如果文件不存在则创建新文件。
    • 文件指针位置：文件末尾。
    • 如果文件无法打开，会引发 PermissionError 异常。
  4. “x” (Exclusive Creation Mode):
    • 用途：以排它性创建模式打开文件，只能用于创建新文件，如果文件已存在则会引发 FileExistsError 异常。
    • 文件指针位置：文件开头。
    • 如果文件无法创建，会引发 PermissionError 异常。
  5. “t” (Text Mode):
    • 用途：以文本模式打开文件，默认模式，用于读取或写入文本文件。
    • 文件指针位置：取决于具体使用情况。
  6. “b” (Binary Mode):
    • 用途：以二进制模式打开文件，用于读取或写入二进制文件（例如图像、视频等）。
    • 文件指针位置：取决于具体使用情况。
  7. “+” (Update Mode):
    • 用途：以更新模式打开文件，可读写文件，用于在同一文件中进行读取和写入操作。
    • 文件指针位置：取决于具体使用情况。

这些模式可以组合使用，例如 "rt" 表示以文本模式打开文件以供读取。根据具体的需求和操作，选择适合的打开模式可以确保对文件的正确读取和写入操作。记住，在操作完文件后，应当正确地关闭文件，以释放资源并确保数据的完整性。

数据结构篇：

Linked List(链表)

Linked List（链表）是一种常见的数据结构，用于存储和组织数据。它由一系列节点（Node）组成，每个节点包含两个部分：数据和指向下一个节点的指针。

Linked List 中的节点通过指针链接在一起，形成一个链式结构。每个节点只知道下一个节点的位置，而不需要在内存中连续存储。这使得插入和删除节点的操作更加高效灵活，但访问节点的随机访问则相对较慢。打个比方：你去医院排队挂号看病，是不是只要拿了那个号码牌就可以随便乱跑了，只要在医生念到你名字的时候再回来就行了。同理，你在内存里面也不需要是连续的，因为你身上有一张号码牌！

比方：可恶，我又被打了！。

  1. 插入和删除（在链表头部/尾部）：

    • 插入操作：可以在链表的头部或尾部插入一个新节点。插入新节点时，只需修改节点之间的指针关系，无需移动其他节点。因此，插入操作的时间复杂度为 O(1)。
    • 删除操作：可以从链表的头部或尾部删除节点。同样，只需修改节点之间的指针关系即可。删除操作的时间复杂度也为 O(1)。

  2. 双向链表（Doubly Linked List）：

    • 双向链表是一种改进的链表结构，在每个节点中维护两个指针，一个指向前一个节点，一个指向后一个节点。这样，每个节点既可以向前遍历，也可以向后遍历。
    • 双向链表支持更多的灵活性和功能。例如，可以在常数时间内删除节点的前驱或后继节点，而不需要遍历整个链表。
    • 双向链表相对于单向链表的缺点是占用更多的内存空间，因为每个节点需要存储额外的指针。

总结来说，链表是一种动态数据结构，支持高效的插入和删除操作，但对于随机访问则效率较低。双向链表是链表的一种扩展形式，提供了更多功能，但相对占用更多内存。选择使用链表还是双向链表取决于具体的应用需求和操作的频率。

时间复杂度：

以下是链表和双向链表中常见操作的时间复杂度

链表（Linked List）操作的时间复杂度：

  • 插入（在头部或尾部插入）：O(1)
  • 删除（在头部或尾部删除）：O(1)
  • 搜索（按值搜索）：O(n)
  • 访问（按索引访问）：O(n)
  • 获取长度：O(n)

双向链表（Doubly Linked List）操作的时间复杂度：

  • 插入（在头部或尾部插入）：O(1)
  • 删除（在头部或尾部删除）：O(1)
  • 搜索（按值搜索）：O(n)
  • 访问（按索引访问）：O(n)
  • 获取长度：O(n)

需要注意的是，链表和双向链表在插入和删除操作上的时间复杂度是 O(1)，因为只需要修改节点之间的指针关系，而不需要像数组一样进行元素的搬移。然而，搜索、访问和获取长度等操作需要遍历链表中的节点，因此时间复杂度为 O(n)，其中 n 是链表中的节点数。

Maps and Dictionaries（映射和字典）

Maps and Dictionaries（映射和字典）是用于存储键值对的数据结构，它们提供了高效的查找和检索功能。在某些编程语言中，python:你直接报我身份证号码吧。字典是映射的具体实现。

Hash Table（哈希表）

Hash Table（哈希表）是一种常见的数据结构，用于实现字典和映射。它基于哈希函数将键映射到存储桶（buckets）中，从而实现快速的查找和插入操作。

Hash Function（哈希函数）

Hash Function（哈希函数）是一种将输入数据映射到固定长度值的函数。它将任意大小的数据转换为固定大小的哈希值，通常是一个整数或字符串。好的哈希函数应该具有均匀分布、高效计算和最小冲突的特性。

Load Factor（负载因子）

Load Factor（负载因子）是衡量哈希表填满程度的度量。它表示哈希表中已存储元素的数量与哈希表大小的比值。负载因子可以帮助我们了解哈希表的空间利用率和性能情况。通常，负载因子越大，哈希表越满，可能会导致冲突增加，从而影响查找和插入操作的性能。
