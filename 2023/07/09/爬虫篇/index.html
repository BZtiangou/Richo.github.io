爬虫篇

前景知识：

静态网站和动态网站
网站可以根据数据的呈现形式分为动态网站和静态网站:静态网站:静态网站是指全部由HTML 标准通用标记语言的子集)代码格式页面组成的网站，所有的内容包含在网页文件中。所有的数据呈现在页面上，不需要和数据库做交互。
动态网站:指网站内容可根据不同情况动态变更的网站，一般情况下动态网站通过和数据库交互来实现，数据会随着数据库进行更新。

动态网页中的数据加载

服务器在想浏览器或爬虫程序发送数据时，将html代码和数据单独发送，浏览器在接收时通过html中的JavaScript代码渲染数据。

html中的JavaScript代码在获取到服务器传入的数据后会进行渲染
数据会有单独的url，格式多为json格式。

HTML

HTML由各种不同含义的标签元素组成 (<标签/标签>)

至少得知道以下标签：

  • p:段落标签，一般用来存储文本信息，如影评、文档等数据
  • a:超链接标签，用来存放网页中其他页面的连接，在爬取数据时，如果想要获取整站的数据需要，收集网站中的a标签，然后逐个去访问。
  • div:区隔标记，没有具体含义。代表了网页中某一块区域。是构造网页时最为常用的一种标签
  • img:图片标签，用来存放网站中的图片数据

层叠样式表(CSS)
样式定义如何显示HTML 元素，在网页中为了样式的统一，会使某些数据呈现相同格式，爬虫程序则可以根据这些样式信息去找数据。

可以定义元素的位置和是否显示，在一些网站中为了避免自己的数据被轻易获取，会使用CSS对数据进行隐藏或修改。

JS：

JavaScript是一种脚本语言，主要用于Web。它用于增强 HTML页面，通常可以嵌入 HTML 代码中。JavaScript 是一种解释型语言。因此，它不需要编译。JavaScript 以交互式和动态的方式呈现网页。这允许页面对事件做出反应，展示特殊效果，接受可变文本验证数据，创建 cookie，检测用户的浏览器等。

JavaScript的作用
JS在网站中可以充当数据的“载体”使前后端进行数据交互。服务器在向浏览器传递数据时可以选择使用JavaScript将数据渲染在html中.
一些网站会在请求响应的过程中使用JavaScript生成额外的参数来阻止爬虫程序的访问。
JavaScript可以实现Ajax(Asynchronous JavaScript And XML异步JavaScript 及XML)

请求方法：
get:从指定的资源请求数据

查询字符串(名称/值对)是在get请求的 URL中发送的。

get请求有长度限制。

post:向指定的资源提交要被处理的数据
查询字符串(名称/值对)是在 post 请求的 HTTP 消息主体中发送的

post 请求对数据长度没有要求

返回值：

200 成功

300+ 重定向(访问到一个不存在的东西，会给你跳转到一个已知的页面)

400+ 页面不存在

500+服务器炸了

网页的cookie:

网页的 Cookie 是一种在用户计算机上存储小型文本文件的技术，用于在用户浏览网页时跟踪和存储有关用户的信息。它是通过网页服务器发送给浏览器并存储在用户计算机上的。每次用户访问同一个网站时，浏览器会将存储的 Cookie 信息发送回服务器，以便服务器根据该信息执行相应的操作或提供个性化的体验。

Cookie有两种类型：会话Cookie和持久Cookie。会话Cookie仅在用户访问网站期间存在，并在关闭浏览器后被删除。持久Cookie存储在用户计算机或设备上，可以在多个会话之间保持有效。持久Cookie具有特定的过期日期或持续时间，直到过期或被用户手动删除。

Cookie 可以包含各种信息，例如用户的首选项、会话状态、登录凭证等。它们通常用于以下目的：

  1. 会话管理：Cookie 可以跟踪用户的会话状态，例如在用户登录网站时创建一个会话 Cookie，在用户退出或关闭浏览器时销毁该会话。
  2. 个性化体验：网站可以使用 Cookie 存储用户的首选项，如语言选择、主题设置等，以便在用户下次访问时提供个性化的体验。
  3. 跟踪和分析：Cookie 可以用于跟踪用户的行为和活动，例如记录用户访问的页面、点击的链接等，以便网站分析用户行为并改进用户体验。
  4. 广告定向：一些 Cookie 用于跟踪用户的兴趣和偏好，以便向用户提供相关的广告内容。
  5. 跨站点请求伪造（CSRF）防护：Cookie 可以用于防止跨站点请求伪造攻击，通过在请求中包含令牌或验证码来验证请求的合法性

阿贾克斯请求：观察网页url是否改变，若不变但页面刷新，则是。用处：对其数据包指定的url进行请求，页面也有可能是动态加载出来的，也就是阿贾克斯请求：他不是由搜索的url请求出来

urllib模块简介
urilib是Python中内置的HTTP请求库，它提供了一系列用于操作URL的功能，其中包括URL发送请求、解析、对请求异常时的处理和对于robots文件的处理。因为博主不常用，所以不详细介绍

Requests模块：

Requests:让HTTP服务人类
Requests是基于urllib库，使用Python实现的简单易用的HTTP库使用起来比urllib简洁很多。
Requests在Python2 和Python3中通用，方法完全一样

requests.get(url，[headers]，[ proxies] [params,] *args): 模拟浏览器向目标主机发送get请求，返回response对象。

  • url:服务器对应的地址
    -headers: HTTP请求中headers字段。
  • proxies:设置代理iP地址。
    params:查询字符串。

timeout: 超时参数，用于设置请求时间，当超过时间仍未获得响应时，会抛出异常(可以结合retrying模块一起使用)

使用requests发送请求后会获取一个响应对象response。响应信息:

  • response.headers 获取响应头信息
    response.status_code 响应状态码- response.encoding 获取文本编码格式响应内容
  • response.text 响应体 str类tnresponsejson（）解码json数据

响应内容(二进制)

  • response.content 获取二进制响应内容 (推荐)原始响应内容
  • response.raw 获取来自服务器的原始套接字响应(需要在请求时指定参数stream=True)
    其他:
    -response.raise for status) 响应码不为200时抛出异常

Requests设置cookie信息
使用requests处理cookie有三种方法

  • 把cookie字典放传给请求方法的cookies参数接收。cookies ={“cookie的name”.”cookie的value”requests.get(url,headers=headers,cookies=cookie dict
  • 使用requests.session处理cookie
  • 在headers字段中添加cookie字符串headers =[“cookie的name”.”cookie的value”

scrapy爬虫框架：

spider:利用蜘蛛将爬取的目标信息打包成一个块，其中要创建item的实例。

items:物块打包的时候需要创建的实例，它包含不同类型物块的类，都继承自django_model的基类。

django_model:存储了数据库的一些相关字段，而model的基类又继承自BaseModel

BaseModel：放在models.py 属于常用的一些工具。它属于自定义类的一种，可以包括创建时间，更新时间和附带的属性

pipelines: 将块送到管道中过滤，清洗后放入数据库

其中的piplines需要在spider里面利用custom_settings 中的”ITEM_PIPELINES”字典指定对应的管道以及优先级。一个空的init.py文件可以作为模块的起始位置。
